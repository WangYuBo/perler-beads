# 拼豆项目协作规范（精简版）

## 提需求格式

直接发文字描述，**分段写**，每段一个需求点：

```
1. 人工编辑加撤回键、取色、画线功能（现在一个个点太累）
2. 本地保存历史记录，刷新不丢失
3. 杂色剔除能力再升级，现在颜色种类还是太多
```

**着急的加前缀**：【着急】、【有空时】

---

## 开发反馈

我会直接在需求下方回复，用状态标签：

| 状态 | 含义 |
|------|------|
| ✅ **可行** | 确认做，给大概时间 |
| 🚧 **开发中** | 正在做 |
| ✔️ **已上线** | 代码已提交 |
| ❌ **暂不做** | 说明原因 |
| ❓ ** unclear** | 需要补充细节 |

**示例：**

> 1. 人工编辑加撤回键、取色、画线
> ✅ **可行** | 预计 2-3 天，画线功能需要确认：是按住拖拽连续画，还是两点连线？

---

## 改需求规则

**不要删原文**，用删除线划掉 + 补充：

```
3. ~~杂色剔除能力再升级~~ → 不只是杂色，整体颜色合并力度要大一些，控制在 15 色以内
```

## 文档分区（飞书里用标题）

```
📋 待处理    ← 新需求放这里
🚧 开发中    ← 我正在做的
✔️ 已完成    ← 已交付的
```

新需求默认放「📋 待处理」最上面，我来排序。
